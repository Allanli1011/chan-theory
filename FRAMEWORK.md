# 缠论技术分析框架（从《教你炒股票》原文提炼）

本文档把缠中说禅《教你炒股票》系列博客中的技术分析体系，提炼为一套**可计算的几何算法**，
并标注每条规则的课程出处（语料见 [`corpus/`](corpus/)）。缠论本质是一套关于走势图形的几何理论：

> “本ID的线段是可以最精确定义的……本质上是一套几何理论，其有效性就如同几何一般。”——课62

完整递归流水线：

```
原始K线 ──包含处理──▶ 合并K线 ──▶ 分型 ──▶ 笔 ──▶ 线段 ──▶ 走势中枢 ──▶ 走势类型
                                                                  │
                                                       MACD背驰 ──┴──▶ 三类买卖点
```

---

## 1. 走势分类与两条基本原理（课17）

> “任何级别的所有走势，都能分解成趋势与盘整两类，而趋势又分为上涨与下跌两类。”——课17

- **基本原理一（走势终完美）**：任何级别的任何走势类型终要完成。
- **基本原理二**：任何级别任何完成的走势类型，必然包含一个以上的走势中枢。
- **走势分解定理二**：任何级别的任何走势类型，都至少由三段以上次级别走势类型构成。

→ 代码：[`models.TrendType`](chanlun/models.py)，[`signals.classify_trend`](chanlun/signals.py)

## 2. K线包含处理（课62 / 课65）

> “向上时，把两K线的最高点当高点，而两K线低点中的较高者当成低点……向下时，把两K线的最低点当低点，而两K线高点中的较低者当成高点。”——课62

- 方向由“被包含前的走势方向”决定（`last.high > prev.high` 视为向上）。
- 须遵守**顺序原则**（课65）：先合并前两根为新K线，再与第三根比较，逐根处理；包含关系不满足传递律。

→ 代码：[`cmerge.merge_klines`](chanlun/cmerge.py)

## 3. 分型（课62）

> “第二K线高点是相邻三K线高点中最高的，而低点也是相邻三K线低点中最高的……叫顶分型。”——课62

- 顶分型：中间K线 `high` 与 `low` 均为相邻三根（合并K线）之最高；底分型反之。
- 顶分型之顶、底分型之底，简称“顶/底”。

→ 代码：[`fractal.find_fractals`](chanlun/fractal.py)

## 4. 笔（课62 / 课65）

> “两个相邻的顶和底之间构成一笔……一定是相邻的顶和底。”——课62
> “在实际分析中，都必须要求顶和底之间都至少有一K线当成一笔的最基本要求。”——课62

- 一顶一底交替；顶底**不得共用K线**（违反结合律），且二者之间**至少有1根独立K线**
  （课62“图5”——一笔的最基本图形）。等价于：顶、底分型中心合并K线下标之差 `>= 4`
  （含两端至少 5 根合并K线，即常说的“标准笔/老笔”；放宽到 3 即“新笔”，可配置 `min_dist`）。
- 两个同类分型之间若无其它分型，则较弱的一个忽略（课65）。

> “上升的一笔……一定是底分型+上升K线+顶分型；下降的一笔，就是顶分型+下降K线+底分型。”——课62

→ 代码：[`bi.build_bis`](chanlun/bi.py)

## 5. 线段——特征序列法（课65 / 课67）

> “线段至少有三笔……这三笔必须有重叠的部分。”——课65

**笔破坏定义（课65）**：以向上笔开始的线段，分型序列 d1g1d2g2…，若存在 `j>=i+2` 使 `dj<=gi`，则向上线段被笔破坏。

**特征序列划分标准（课67，可操作的精确标准）**：

> “以向上笔开始的线段……序列 X1X2…Xn 称为特征序列……特征序列两相邻元素间没有重合区间，称为该序列的一个缺口。”——课67

- 以向上笔开始的线段，特征序列取其中的**向下笔**，只考察**顶分型**；以向下笔开始者取向上笔，只考察底分型。
- 特征序列须先做**非包含处理** → 标准特征序列。
- **第一种情况**：顶分型中第一、第二元素间**无缺口** ⟹ 线段在该顶分型高点处结束。
- **第二种情况**：第一、第二元素间**有缺口** ⟹ 须“从该分型最高点开始的向下一笔……的特征序列出现底分型”才确认结束。

> “线段破坏的充要条件，就是被另一个线段破坏。”——课65

→ 代码：[`segment.build_segments`](chanlun/segment.py)（含缺口两种情况、有界确认）

## 6. 走势中枢（课17 / 课18）

> “某级别走势类型中，被至少三个连续次级别走势类型所重叠的部分，称为缠中说禅走势中枢。”——课17

**精确计算（课18）**：连续三个次级别走势 A、B、C，高低点 (a1,a2)(b1,b2)(c1,c2)，

> “中枢的区间就是（max(a2,b2,c2)，min(a1,b1,c1)）”——课18

即 `ZG = min(各高点)`、`ZD = max(各低点)`，需 `ZG > ZD`；另记 `GG=max(高点)`、`DD=min(低点)`。

- **盘整**：走势类型只含一个中枢；**趋势**：至少两个**依次同向且互不重叠**的中枢（课17/课18）。
- **中枢定理三（破坏）**：

> “某级别中枢的破坏，当且仅当一个次级别走势离开该中枢后，其后的次级别回抽走势不重新回到该中枢内。”——课18

本实现提供两种中枢分解视角：`extension` 以“公共重叠区间”判据扩展/终止中枢；
`same_level` 则按同级别分解，只用首个三段重叠定义中枢，后续段不继续吞入同一中枢。
趋势分类要求相邻中枢全程同向且互不重叠，否则仍归为盘整。

→ 代码：[`zhongshu.find_zhongshus`](chanlun/zhongshu.py)

## 7. 背驰与 MACD 辅助（课24）

> “用MACD判断背驰，首先要有两段同向的趋势……把这三段分别称为A、B、C段……C段的走势类型完成时对应的MACD柱子面积（向上的看红柱子，向下看绿柱子）比A段对应的面积要小，这时候就构成标准的背弛。”——课24

- **背驰-买卖点定理**：任一背驰都必然制造某级别的买卖点；任一级别的买卖点都必然源自某级别走势的背驰。
- 技巧：“柱子伸长的力度变慢时，把已经出现的面积乘2”近似该段面积（课24）。
- **盘整背驰**：盘整中 C 段面积小于 A 段亦有效；若 C 上破中枢但面积更小，回跌不回中枢即构成**第三类买点**（课24）。

→ 代码：[`macd.MacdHelper`](chanlun/macd.py)（红/绿柱面积、DIF极值），背驰判定见 [`signals`](chanlun/signals.py)

## 8. 三类买卖点（课17 / 课24 / 中枢定理三）

| 类型 | 定义 | 出处 |
|------|------|------|
| **第一类** | 趋势背驰的转折点：下跌背驰→一买（底）、上涨背驰→一卖（顶） | 课17/课24 |
| **第二类** | 一类点后**第一次次级别回抽不破前低/前高** | 课17“买卖点定律一” |
| **第三类** | 走势离开中枢后，**回抽不重新回到中枢区间** [ZD,ZG]（上破不回 ZG 之下→三买；下破不回 ZD 之上→三卖） | 课18 中枢定理三 / 课24 |

> “任何级别的第二类买卖点都由次级别相应走势的第一类买点构成。”——课17（买卖点定律一）
> “任何的下跌转折都是由某级别的第一类买点构成的。”——课17（趋势转折定律）

→ 代码：[`signals.find_buy_sell_points`](chanlun/signals.py)

## 9. 级别与递归（课17 / 课65）

> “这次级别不能无限下去……对最后不能分解的级别，其缠中说禅走势中枢就……定义为至少三个该级别单位K线重叠部分。”——课17

- K线→笔→线段→（更高级别走势）逐级递归；不同周期图（1F/5F/30F/日/周）对应不同“显微镜倍数”。
- 实现以**笔**为次级别走势单元构建笔中枢、识别本级别买卖点；**线段**用于更高级别结构与线段中枢。

→ 代码：[`analyzer.analyze_multi_level`](chanlun/analyzer.py)（重采样到更高周期分别分析）

## 与原文的工程取舍说明

缠论的笔/线段在边界情形（缺口确认、中枢延伸 vs 新中枢、次级别回抽级别）存在公认的实现分歧，
缠在课71/77/78 亦多次“再分辨”。本实现：

1. 笔：默认“标准笔”（中心距 `>=4`，可配 `min_dist`），用走势腿跟踪法过滤次级别噪声并保留真实极值。
2. 线段：以课67特征序列法为准，第二种情况（缺口）采用**有界确认**（先创新极值则该分型作废）。
3. 中枢：默认用“公共重叠”判据延伸，也支持 `same_level` 同级别分解；`ZG/ZD` 含义同课18。
4. 线段：线段高低点统计其内部全部笔；可能未完成的末线段会保留展示，但不参与更高级别中枢。
5. 背驰：以**笔**为最小比较单元，但必须关联并离开相应中枢，避免把无中枢上下文的局部波动误标为一类买卖点。

以上取舍均在代码 docstring 中标注课程出处，便于对照原文调整。
